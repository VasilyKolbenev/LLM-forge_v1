# Pulsar AI -- Руководство пользователя

---

## 1. Введение

### Что такое Pulsar AI

Pulsar AI -- платформа для полного цикла работы с языковыми моделями: от подготовки данных и обучения (fine-tuning) до оценки, развертывания и мониторинга. Платформа предоставляет веб-интерфейс и CLI для управления всеми этапами.

### Ключевые возможности

- **Эксперименты** -- запуск и отслеживание fine-tuning задач (SFT, DPO)
- **Датасеты** -- загрузка, предпросмотр и управление данными (текст + мультимодальные)
- **Workflows** -- визуальный конструктор DAG-пайплайнов с 30+ типами нод
- **Agent Office** -- 3D-визуализация агентных пайплайнов в изометрическом стиле
- **Мониторинг** -- метрики CPU, RAM, GPU в реальном времени
- **Compute** -- управление локальными и удаленными GPU-серверами
- **Prompt Lab** -- версионирование, тестирование и сравнение промптов
- **Agent Chat** -- чат с обученными моделями
- **Traces** -- просмотр execution traces агентов, обратная связь, экспорт в датасет
- **Agent Eval** -- оценка агентов по тестовым наборам с метриками качества
- **OpenClaw** -- runtime для AI-агентов с песочницей и governance
- **Lifecycle** -- визуализация lineage: dataset -> experiment -> model -> deployment
- **Benchmarks** -- бенчмаркинг моделей (tokens/sec, TTFT, VRAM, perplexity)
- **Settings** -- API-ключи, workspaces, approvals, audit log
- **Admin Panel** -- управление пользователями, здоровье системы, конфигурация

### Для кого предназначена платформа

- ML-инженеры, занимающиеся fine-tuning LLM
- Команды, работающие с AI-агентами
- DevOps/MLOps-специалисты, управляющие compute-ресурсами
- Руководители ML-команд, которым нужен контроль и governance

## 2. Начало работы

### Регистрация и вход

1. Откройте Pulsar AI в браузере.
2. На экране входа выберите вкладку **Register** (переключатель вверху формы).
3. Заполните поля:
   - **Name** -- ваше имя
   - **Email** -- корпоративная почта
   - **Password** -- минимум 6 символов
4. Нажмите **Create Account**.
5. После регистрации вы будете перенаправлены на Dashboard.

Для последующих входов используйте вкладку **Sign In** и введите email + пароль.

### MFA (двухфакторная аутентификация)

Если администратор включил MFA:

1. После ввода email и пароля появится экран **Two-Factor Authentication**.
2. Откройте приложение-аутентификатор (Google Authenticator, Authy и т.д.).
3. Введите 6-значный TOTP-код в поле **Verification Code**.
4. Нажмите **Verify**.
5. Если вы потеряли доступ к аутентификатору, используйте backup-код в том же поле.

Кнопка **Back to login** возвращает на экран ввода пароля.

### SSO вход (OIDC)

Если настроен SSO-провайдер (Okta, Azure AD и т.д.):

1. На экране входа появится кнопка **Sign in with {ProviderName}** над формой.
2. Нажмите на нее -- вы будете перенаправлены на страницу провайдера.
3. Пройдите аутентификацию на стороне провайдера.
4. После успешной аутентификации вы вернетесь в Pulsar AI на Dashboard.
5. Если у провайдера включен MFA, после возврата может появиться экран ввода TOTP-кода.

### Интерфейс

**Sidebar (боковая панель)** -- слева, ширина 224px. Содержит:

- Логотип Pulsar AI и подпись "Fine-tuning Platform"
- Навигационные ссылки на все разделы (15 пунктов)
- Пункт **Admin** виден только пользователям с ролью `admin`
- **Resource Bar** внизу -- мини-индикаторы CPU/GPU загрузки в реальном времени
- Блок пользователя: имя, роль, кнопка **Logout**

Активный пункт меню подсвечивается цветом primary. Интерфейс использует темную тему по умолчанию.

## 3. Dashboard

Путь: `/dashboard`

Dashboard -- стартовый экран после входа. Содержит обзор всей платформы.

### Карточки статистики (верхняя строка)

Четыре карточки в одну строку:

- **Experiments** -- общее количество, сколько running / completed

- **Prompts** -- количество версионированных промптов

Три карточки ниже:

- **GPU** -- название GPU, текущий % утилизации, использование VRAM (X.X/Y.Y GB), мини-график за 2 минуты
- **CPU** -- количество ядер, текущий % загрузки, RAM (X.X/Y.Y GB), мини-график за 2 минуты
- **Quick Actions** -- быстрые ссылки:
  - New Experiment
  - Build Workflow
  - New Prompt
  - Add Compute

### Таблица последних экспериментов

Внизу Dashboard -- таблица **Recent Experiments** (до 8 записей) с колонками:

| Колонка | Описание |
|---------|----------|
| Name | Имя эксперимента (ссылка) |
| Status | running / completed / failed / queued |
| Task | SFT или DPO |
| Loss | Финальный loss (или "--") |
| Created | Дата создания |

Ссылка **View all** ведет на полный список экспериментов.

## 4. Эксперименты

### Создание нового эксперимента

Путь: `/new`

Мастер создания в 4 шага. Индикатор шагов отображается вверху: **Setup -> Dataset -> Training -> Progress**.

**Шаг 1: Setup**

- **Experiment Name** -- уникальное имя (например, `my-classifier`)
- **Task** -- выбор стратегии обучения:
  - `SFT (Supervised Fine-Tuning)` -- обучение на парах instruction/response
  - `DPO (Direct Preference Optimization)` -- обучение на предпочтениях
- **Base Model** -- выбор базовой модели через Model Selector (например, `Qwen/Qwen2.5-3B-Instruct`)

Нажмите **Next: Dataset** для перехода к следующему шагу.

**Шаг 2: Dataset**

- **Select Dataset** -- выпадающий список загруженных датасетов с указанием количества строк
- Альтернативно: введите путь вручную в поле ниже (например, `data/train.csv`)

**Шаг 3: Training Parameters**

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| Learning Rate | 2e-4 | Скорость обучения |
| Epochs | 3 | Количество эпох |
| Batch Size | 1 | Размер батча |
| Gradient Accumulation | 16 | Накопление градиентов |
| Max Seq Length | 512 | Максимальная длина последовательности |

Нажмите **Start Training** для запуска.

**Шаг 4: Progress**

После запуска отображается:

- **Status** -- текущий статус (running / completed / error)
- Три карточки метрик: Step, Loss, Epoch
- **Loss Chart** -- график loss в реальном времени (обновляется через SSE)

### Просмотр экспериментов

Путь: `/experiments`

Таблица всех экспериментов с колонками:

| Колонка | Описание |
|---------|----------|
| Checkbox | Выбор для сравнения |
| Name | Кликабельное имя (открывает детали) |
| Status | running / completed / failed / queued с цветным бейджем |
| Model | Базовая модель |
| Strategy | SFT / DPO / GRPO |
| Loss | Финальный loss |
| Accuracy | Overall accuracy в % |
| Trainable | % обучаемых параметров |
| Duration | Длительность обучения в минутах |
| Created | Дата и время |
| Delete | Кнопка удаления (иконка корзины) |

Автообновление: если есть running-эксперименты, таблица обновляется каждые 3 секунды.

### Детали эксперимента

Клик на имя эксперимента открывает панель **ExperimentDetail** с подробной информацией, включая артефакты, метрики и конфигурацию.

### Сравнение экспериментов

1. Отметьте чекбоксами 2 или более экспериментов.
2. Нажмите кнопку **Compare (N)** в правом верхнем углу.
3. Откроется панель **Comparison** с JSON-представлением различий между экспериментами.

## 5. Датасеты

Путь: `/datasets`

### Загрузка

Зона загрузки в верхней части страницы поддерживает drag & drop:

1. Перетащите файл в зону с иконкой Upload, или кликните для выбора файла.
2. Поддерживаемые форматы: **CSV**, **JSONL**, **Parquet**, **Excel (.xlsx)**
3. Индикатор "Uploading..." показывает процесс загрузки.

### Таблица датасетов

| Колонка | Описание |
|---------|----------|
| Name | Имя файла |
| Type | **Text** или **Multimodal** (с иконкой Image, фиолетовый бейдж) |
| Format | CSV / JSONL / PARQUET / XLSX |
| Rows | Количество строк |
| Size | Размер в KB |
| Actions | Preview (иконка глаза) и Delete (корзина) |

### Мультимодальные датасеты

Система автоматически определяет мультимодальные датасеты по наличию колонок: `image`, `image_path`, `image_url`, `img`, `img_path`, `photo`. Такие датасеты помечаются бейджем **Multimodal** с иконкой Image.

### Предпросмотр

Нажмите иконку глаза (Eye) -- откроется панель **Preview**:

- Заголовок показывает общее количество строк
- Таблица с данными (первые строки)
- Колонки с изображениями помечены иконкой Image
- Значения image-колонок отображаются в фиолетовом моноширинном стиле
- Кнопка **Close** закрывает предпросмотр

## 6. Workflows (визуальный конструктор)

Путь: `/workflows`

### Концепция

Workflows -- визуальный DAG-конструктор пайплайнов. Вы перетаскиваете ноды из палитры слева на канвас и соединяете их ребрами, формируя граф обработки данных.

### Toolbar (панель инструментов)

Вверху страницы:

- **Поле имени** -- название workflow (редактируемое)
- **View Toggle** -- переключатель режимов просмотра:
  - **Flow** (DAG) -- классический граф
  - **Sims** (Office) -- 3D-изометрическая визуализация Agent Office
  - **Split** -- оба режима рядом (50/50)
- **Banking Template** -- загрузка готового шаблона банковского пайплайна
- **Load** -- открыть сохраненный workflow
- **Save** -- сохранить текущий
- **Replay** -- воспроизведение trace с анимацией
- **Run** -- запуск пайплайна
- **Trash** -- очистить канвас

### Палитра нод (Node Palette)

Левая панель (ширина 224px) с группами нод. Перетаскивайте ноды на канвас.

**Data:**
- Data Source -- источник данных
- Model -- модель
- Prompt -- промпт
- Splitter -- разделение данных

**Training:**
- Training -- запуск обучения

**Evaluation:**
- Evaluation -- оценка модели
- Condition -- условный переход
- LLM Judge -- оценка через LLM
- A/B Test -- A/B тестирование

**Agent:**
- Agent -- AI-агент
- RAG -- retrieval-augmented generation
- Router -- маршрутизация запросов
- Inference -- инференс модели
- Data Gen -- генерация данных

**Protocols:**
- MCP -- Model Context Protocol
- A2A -- Agent-to-Agent
- Gateway -- API-шлюз

**Safety:**
- Input Guard -- фильтрация входных данных
- Output Guard -- фильтрация выходных данных

**Ops:**
- Cache -- кеширование
- Canary -- канареечное развертывание
- Feedback -- сбор обратной связи
- Tracer -- трассировка

**Closed Loop:**
- Collect Traces -- сбор traces
- Build Dataset -- построение датасета из traces
- Regression Eval -- регрессионная оценка

**OpenClaw:**
- OpenClaw Session -- сессия OpenClaw
- NemoClaw Deploy -- развертывание в песочнице

**Structure:**
- Group -- группировка нод

### Работа с канвасом

1. Перетащите ноду из палитры на канвас.
2. Соедините ноды: перетащите от выходного порта одной ноды к входному порту другой.
3. Кликните на ноду -- справа откроется **Properties Panel** для настройки параметров.
4. Двойной клик на ноду -- откроется **Persona Editor** для настройки аватара и персоны агента.

### Agent Office (3D-визуализация)

Переключите режим на **Sims** или **Split**. Каждая нода отображается как персонаж в изометрическом офисе с аватаром. Доступные персоны: Scientist, Engineer, DevOps, Architect, Security, Observer.

### Replay (воспроизведение trace)

1. Нажмите **Replay** в toolbar.
2. Появится панель управления внизу: Play/Pause, Step Forward, Seek, Speed.
3. Ноды подсвечиваются по мере прохождения trace.
4. Кнопка **Close** выходит из режима Replay.

### Сохранение и загрузка

- **Save** -- сохраняет текущий workflow с именем, нодами и ребрами
- **Load** -- открывает модальное окно **Saved Workflows** со списком:
  - Имя workflow
  - Дата обновления, количество запусков
  - Клик загружает workflow, кнопка корзины удаляет
  - **New Workflow** -- создать новый (очищает канвас)

## 7. Мониторинг

Путь: `/monitoring`

### Обзор

Страница системного мониторинга с метриками в реальном времени. Индикатор подключения в правом верхнем углу: зеленая точка "Connected" или красная мигающая "Reconnecting...".

### Карточки текущих значений

- **GPU Utilization** -- % использования GPU, название GPU
- **VRAM** -- использование видеопамяти в GB
- **CPU** -- % загрузки, количество ядер
- **RAM** -- использование оперативной памяти в GB

### Графики

Все графики обновляются в реальном времени:

- **GPU Utilization** -- area chart, 0-100%, фиолетовый
- **VRAM Usage** -- area chart, 0-max GB, фиолетовый
- **GPU Temperature** -- line chart, 0-100 C, красный
- **Power Draw** -- line chart, в ваттах, желтый
- **CPU Usage** -- area chart, 0-100%, зеленый
- **RAM Usage** -- area chart, 0-max GB, синий

### Multi-GPU

Если в системе несколько GPU, внизу появляется таблица **All GPUs** с колонками: #, Name, Util %, VRAM, Temp, Power.

## 8. Compute

Путь: `/compute`

### Local Machine

Всегда отображается карточка **Local Machine** с зеленым индикатором. Ссылка "View metrics" ведет на страницу Monitoring.

### Добавление удаленного сервера

1. Нажмите **Add Target** в правом верхнем углу.
2. В модальном окне заполните:
   - **Name** -- имя сервера (например, `my-gpu-box`)
   - **Host** -- IP-адрес или hostname (например, `192.168.1.100`)
   - **Port** -- SSH-порт (по умолчанию `22`)
   - **User** -- SSH-пользователь (по умолчанию `ubuntu`)
   - **SSH Key Path** -- путь к SSH-ключу (опционально, например `~/.ssh/id_rsa`)
3. Нажмите **Add Target**.

### Управление серверами

Каждый сервер отображается карточкой с информацией:

- Имя и статус (online/offline/unknown с цветным бейджем)
- Подключение: `user@host:port`
- Информация о GPU: количество, тип, VRAM

Кнопки действий (справа):

- **Test connection** (иконка RefreshCw) -- проверить SSH-подключение. Показывает латентность при успехе.
- **Detect hardware** (иконка MonitorDot) -- определить GPU на удаленной машине. Показывает количество GPU, тип и VRAM.
- **Remove** (иконка Trash2) -- удалить сервер.

## 9. Prompt Lab

Путь: `/prompts`

### Список промптов

Промпты отображаются карточками в сетке (до 3 в строку):

- Имя промпта
- Описание
- Текущая версия (бейдж `vN`)
- Теги (цветные бейджи)
- Дата обновления

Фильтрация по тегам: вверху отображаются все уникальные теги, клик фильтрует список.

### Создание промпта

1. Нажмите **New Prompt**.
2. В модальном окне заполните:
   - **Name** -- имя (обязательно)
   - **Description** -- описание (опционально)
   - **System Prompt** -- текст промпта (обязательно). Используйте `{{variable}}` для переменных.
   - **Tags** -- теги через запятую
3. Нажмите **Create**.

Пример промпта с переменными:
```
You are {{role}}. Help the user with {{task}}.
```

### Редактирование и версионирование

Клик на карточку промпта открывает детальный вид с тремя колонками:

**Prompt Editor** (основная область):
- Текстовое поле с текущим промптом
- Информация о переменных внизу
- Кнопка **Save as vN+1** -- сохранить как новую версию

**Test Prompt** (под редактором):
- Поля для ввода значений переменных
- Кнопка **Render** -- отрендерить промпт с подставленными значениями
- Результат отображается ниже

**Version History** (правая колонка):
- Список всех версий (от новой к старой)
- Клик на версию загружает ее текст в редактор
- Текущая версия выделена цветом primary

### Сравнение версий (diff)

Если промпт имеет больше одной версии:

1. В блоке **Compare Versions** выберите две версии из выпадающих списков.
2. Нажмите **Diff**.
3. Различия отображаются в блоке ниже.

## 10. Agent Chat

Путь: `/agent`

### Использование

1. Убедитесь, что агент-сервер запущен (`pulsar agent serve`).
2. Введите сообщение в поле внизу экрана.
3. Нажмите Enter или кнопку Send.
4. Ответ появится в чате. Если агент вернул `trace_id`, под ответом появятся кнопки обратной связи (Feedback Buttons).

### Особенности

- Сообщения пользователя -- справа, с фоном primary
- Ответы агента -- слева, с рамкой
- Индикатор "Thinking..." отображается во время ожидания ответа
- Ошибки подключения отображаются как сообщение агента с текстом "Error: ..."

## 11. Traces

Путь: `/traces`

### Обзор

Страница для просмотра execution traces агентов. Вверху -- статистика:

- **Total Traces** -- общее количество
- **With Feedback** -- количество с обратной связью
- **Avg Rating** -- средний рейтинг
- **Error Rate** -- процент ошибок

### Фильтрация

Доступные фильтры:

- **From / To** -- диапазон дат
- **Model** -- фильтр по модели (например, `gpt-4`)
- **Status** -- All / Success / Error
- **Min Rating** -- минимальный рейтинг (0-1)

### Таблица traces

| Колонка | Описание |
|---------|----------|
| Checkbox | Выбор для экспорта в датасет |
| Time | Дата и время |
| Query | Запрос пользователя (обрезается до 60 символов) |
| Model | Модель |
| Status | success / error с цветным бейджем |
| Rating | Thumbs up / Thumbs down или "--" |
| Latency | Задержка в мс |
| Tokens | Количество использованных токенов |

### Детализация trace

Клик на строку раскрывает подробности:

- **User Query** и **Response** -- рядом друг с другом
- **Trace Timeline** -- пошаговое выполнение:
  - Thought (llm_response) -- синий
  - Action (tool_call) -- желтый
  - Observation -- зеленый
  - Final Answer -- фиолетовый
- **Feedback** -- кнопки thumbs up/down, история отзывов

### Экспорт traces в датасет

1. Отметьте чекбоксами нужные traces.
2. Нажмите **Build SFT** или **Build DPO** в правом верхнем углу фильтров.
3. В модальном окне:
   - **Dataset Name** -- имя датасета
   - **Format** -- SFT или DPO
   - Количество выбранных traces
4. Нажмите **Build Dataset**.
5. Результат покажет путь к файлу и количество примеров.

Пагинация: кнопки **Previous / Next** внизу, по 20 записей на страницу.

## 12. Agent Eval

Путь: `/agent-eval`

### Запуск оценки

1. Нажмите **Run Eval**.
2. В модальном окне:
   - **Suite** -- выберите evaluation suite из списка (название и количество кейсов)
   - **Scoring Mode** -- режим оценки:
     - `Exact Match` -- точное совпадение
     - `Contains` -- содержит ожидаемый ответ
     - `LLM Judge` -- оценка через LLM
   - **Agent Config (JSON)** -- конфигурация агента в формате JSON
3. Нажмите **Run Eval**.

### Таблица отчетов

| Колонка | Описание |
|---------|----------|
| Suite | Имя набора тестов |
| Model | Модель |
| Date | Дата и время |
| Success Rate | Процент успеха (прогресс-бар + число) |
| Avg Score | Средний score |
| Latency | Средняя задержка в мс |
| Cost | Стоимость в $ |
| Tokens | Общее количество токенов |

Клик на строку раскрывает:

- **Summary Stats**: Success Rate, Avg Score, Avg Latency, Tools Accuracy

2. Кликните на два отчета для выбора (A и B).

   - **Winner** -- какой отчет лучше (или Tie)

   - **Score Delta** -- разница в score
   - **Latency Delta** -- разница в задержке
   - **Cost Delta** -- разница в стоимости

Зеленый цвет -- улучшение, красный -- ухудшение.

## 13. OpenClaw

Путь: `/openclaw`

### Что такое OpenClaw

OpenClaw -- runtime для запуска AI-агентов с поддержкой sandbox-изоляции и governance. Три вкладки: Sessions, Deployments, Governance.

### Индикатор здоровья

В правом верхнем углу -- статус OpenClaw runtime: зеленый "healthy" или красный "unavailable".

### Sessions (сессии агентов)

Сессия -- запущенный экземпляр агента с определенной моделью и набором инструментов.

**Создание сессии:**

1. Нажмите **New Session**.
2. Заполните:
   - **Name** -- имя агента
   - **Model** -- модель (например, `gpt-4`)
   - **Tools** -- инструменты через запятую
   - **System Prompt** -- системный промпт
3. Нажмите **Create**.

**Действия с сессиями:**

- **Run** (Play) -- запустить сессию
- **Stop** (Square) -- остановить
- **Ingest Traces** (Download) -- загрузить traces сессии в систему
- Клик на строку -- раскрыть trace выполнения

### Deployments (NemoClaw)

NemoClaw -- sandbox-развертывание агента с ограничениями ресурсов.

**Создание deployment:**

1. Нажмите **New Deployment**.
2. Заполните параметры агента (Name, Model, Tools, System Prompt).
3. Настройте sandbox-политику:
   - **Allow Network** -- разрешить сетевой доступ
   - **Allow File Write** -- разрешить запись файлов
   - **Max Memory (MB)** -- лимит памяти (по умолчанию 512)
   - **Max CPU Seconds** -- лимит CPU (по умолчанию 60)
   - **Max Tokens** -- лимит токенов (по умолчанию 4096)
4. Нажмите **Create**.

**Действия:** Stop deployment, просмотр health-статуса.

### Governance (ClawFence)

Вкладка governance содержит:

**Pending Approvals:**
- Список ожидающих одобрения операций
- Каждая запись: session_id, agent_name, requested_at, status
- Кнопки **Approve** / **Reject** для каждой записи

**Audit Events:**
- Журнал событий: timestamp, event_type, session_id, details, actor

## 18. Горячие клавиши и советы

### Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| Enter | Отправить сообщение в Agent Chat |
| Shift+Enter | Новая строка в Agent Chat |
| Перетаскивание | Добавить ноду на канвас Workflows |

### Советы по работе

**Эксперименты:**
- Начинайте с малых значений epochs (1-3) и увеличивайте при необходимости
- Используйте gradient accumulation для имитации больших батчей на GPU с малым VRAM
- Следите за loss-графиком: если loss перестал падать, можно остановить обучение

**Датасеты:**
- Для SFT используйте JSONL с полями `instruction` и `output`
- Для DPO добавьте поле `chosen` и `rejected`
- Проверяйте данные через Preview перед запуском обучения

**Workflows:**
- Используйте шаблон Banking Template как отправную точку
- Группируйте связанные ноды с помощью ноды Group
- Используйте Replay для отладки пайплайна

**Prompt Lab:**
- Всегда используйте переменные `{{variable}}` для динамических частей промпта
- Тестируйте промпты через кнопку Render перед сохранением версии
- Используйте Diff для анализа изменений между версиями

**Traces -> Dataset (closed loop):**
- Собирайте traces с хорошими оценками (thumbs up)
- Экспортируйте в SFT-формат для дообучения
- Это создает замкнутый цикл: agent -> traces -> dataset -> training -> better agent

**Compute:**
- Всегда проверяйте подключение через Test connection перед запуском обучения
- Используйте Detect hardware для автоматического определения GPU

**Benchmarks:**
- Сравнивайте модели по tokens_per_sec для production-сценариев
- Следите за peak_vram_gb для выбора оптимальной модели под ваш GPU
- Perplexity -- ключевая метрика качества модели

**Безопасность:**
- Генерируйте API-ключи в Settings и храните их безопасно
- Используйте Approvals для критических операций в команде
- Регулярно проверяйте Audit Log на подозрительную активность
